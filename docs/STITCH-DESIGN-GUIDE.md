# Luvia Gallery — Google Stitch 前端设计功能指引

> **文档用途**：供 Google Stitch (或任意 AI 设计工具) 理解 APP 功能范围，生成配套 UI 界面设计。
> **视觉风格**：由设计工具自由发挥，本文件仅定义**功能需求与信息架构**。
> **目标平台**：Android (Kotlin 原生重写)
> **文档版本**：v1.0 | 2026-05-10

---

## 一、产品定位

Luvia Gallery 是一款**私人图库应用**，用户连接自托管后端服务器，浏览、收藏、管理个人照片、视频和音频文件。支持多用户、权限隔离和服务器远程管理。

**核心关键词**：私密图库、自托管、多媒体浏览、收藏管理、服务端运维

## 二、界面清单与功能规格

### S1. 启动 / 认证门禁

| 编号 | 界面 | 功能说明 |
|------|------|----------|
| S1.1 | **启动加载屏** (Splash) | 显示品牌 Logo，后台静默检查本地缓存的 Token；自动判断进入登录页或主页 |
| S1.2 | **生物识别锁定屏** (Biometric Gate) | 若用户开启了生物识别锁，APP 从后台恢复时弹出模糊遮罩 + 指纹/面容认证 UI。包含锁定图标、"Luvia 已锁定"标题、描述文案、"点击解锁"按钮。APP 切入后台时自动重新锁定 |
| S1.3 | **登录页** (Login Screen) | 表单包含：用户名输入、密码输入、"登录"按钮。右上角提供语言切换（中/英）和深浅主题切换。底部有"服务器配置"折叠区，展开后可输入后端 URL 地址。品牌展示：Logo + "Luvia Gallery" 标题 + 副标题 |

### S3. 首页 (Discover / Home)

| 编号 | 区域 | 功能说明 |
|------|------|----------|
| S3.1 | **页面标题** | 显示"探索"标题 |
| S3.2 | **Hero 轮播** (Carousel) | 占据页面主要区域的大幅轮播图。支持自动轮播（7秒间隔），Ken Burns 缩放动效。数据源可配置：随机全库 / 仅收藏 / 指定文件夹 / 指定文件。点击任意轮播项进入媒体查看器。宽屏设备（平板）显示多列轮播。支持视频自动静音播放 |

### S7. 媒体查看器 (Media Viewer)

全屏沉浸式覆盖层，覆盖所有其他 UI。

| 编号 | 区域 | 功能说明 |
|------|------|----------|
| S7.1 | **图片模式** | 全屏展示图片，支持双指缩放（最大 5x）、双击放大（3x）、平移拖拽。加载中先显示模糊缩略图占位 |
| S7.2 | **视频模式** | 全屏视频播放器。底部控制栏：播放/暂停、进度条、当前时间/总时长、旋转屏幕按钮。显示视频缩略图作为加载占位 |
| S7.3 | **音频模式** | 沉浸式音频播放界面。大区域音乐图标（专辑封面占位）、歌曲名、模糊背景。底部毛玻璃控制面板：进度条 + 时间、上一首/播放暂停/下一首按钮 |
| S7.4 | **水平滑动切换** | 左右滑动切换上一个/下一个媒体 |
| S7.5 | **顶部控制栏** | 自动 3 秒隐藏。包含：返回按钮、幻灯片播放开关、收藏按钮（红心/空心）、下载按钮、信息面板开关、跳转到所在文件夹按钮 |
| S7.6 | **幻灯片模式** | 开启后自动轮播。图片/音频停留 5 秒，视频播放完毕后自动下一个 |
| S7.7 | **下载功能** | 点击下载按钮，请求存储权限后下载到系统相册。显示下载进度，支持取消。权限被拒时回退到系统分享 |
| S7.8 | **信息面板** | 底部上滑呼出，半透明背景。展示：文件名、修改时间、文件大小、分辨率、文件路径、实时网络速率、视频码率、EXIF 信息（相机型号、ISO、光圈） |
| S7.9 | **手势** | 单击切换控制栏显隐；下滑关闭查看器；上滑呼出信息面板 |

## 四、API 端点清单

所有端点基于用户配置的 `{API_URL}` 前缀。认证端点外的请求需携带 `Authorization: Bearer {token}` 头。

### 认证

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 登录，返回 `{ token, user: { username, isAdmin } }` |

### 媒体浏览

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/scan/results` | 分页获取媒体列表。参数：offset, limit, folder, favorites, random, sort, search, mediaType, excludeMediaType, recursive |
| GET | `/api/file/{id}` | 获取原始文件 |
| GET | `/api/thumb/{id}` | 获取缩略图 |
| GET | `/api/file/{id}/exif` | 获取 EXIF 信息 |
| GET | `/api/library/folders` | 获取文件夹列表。参数：parentPath / favorites=true |

### 操作

| 方法 | 端点 | 说明 |
|------|------|------|
| POST | `/api/favorites/toggle` | 切换收藏状态。Body: `{ id, type: "file"/"folder" }` |
| POST | `/api/file/delete` | 删除文件。Body: `{ filePath }` |
| POST | `/api/folder/delete` | 删除文件夹。Body: `{ path }` |

### 服务器管理（仅管理员）

| 方法 | 端点 | 说明 |
|------|------|------|
| GET | `/api/scan/status` | 获取扫描状态 + 系统统计 |
| POST | `/api/scan/start` | 启动媒体库扫描 |
| POST | `/api/scan/control` | 控制扫描任务。Body: `{ action: "pause"/"resume"/"stop" }` |
| GET | `/api/thumb-gen/status` | 获取缩略图生成状态 |
| POST | `/api/thumb-gen/start` | 启动缩略图生成 |
| POST | `/api/thumb-gen/control` | 控制生成任务 |
| GET | `/api/thumb/smart-results` | 获取缓存校验结果。参数：summary=true |
| POST | `/api/thumb/smart-scan` | 启动缓存校验扫描 |
| POST | `/api/thumb/smart-repair` | 执行智能修复。Body: `{ repairMissing, repairError }` |
| POST | `/api/thumb/extract-dimensions` | 提取缩略图尺寸 |
| POST | `/api/cache/prune` | 清理冗余缓存 |
| GET | `/api/config` | 获取服务器配置（含用户列表） |
| POST | `/api/config` | 更新服务器配置 |
| POST | `/api/users` | 添加用户 |
| POST | `/api/users/{username}` | 编辑用户 |
| DELETE | `/api/users/{username}` | 删除用户 |

## 六、多语言支持

APP 支持中文（zh）和英文（en）两种语言，所有 UI 文案均通过 i18n 系统动态获取。设计时无需硬编码文案，但需预留文本长度的弹性空间（中文通常比英文短 30-50%）。

---

## 七、设计注意事项

1. **深浅双主题**：所有界面必须同时设计 Light 和 Dark 两种模式
2. **安全区域适配**：顶部刘海屏和底部 Home Indicator 区域需要安全边距
3. **响应式布局**：手机（~360dp 宽）和平板（~600dp+ 宽）的自适应。平板上轮播显示多列，网格列数增加
4. **手势交互**：媒体查看器依赖大量手势（缩放、平移、滑动、双击），设计时需考虑手势友好的触摸热区
5. **加载态**：所有数据驱动的界面都需要骨架屏/加载态设计
6. **空状态**：收藏、文件夹等页面需要空状态占位设计
7. **权限区分**：设置页的服务器管理 Tab 仅管理员可见，普通用户看到"拒绝访问"
8. **破坏性操作**：删除、退出登录等操作需要二次确认对话框

> **设计要求**：
> 1. 所有界面需要同时提供 **Light** 和 **Dark** 两种主题方案
> 2. 视觉风格由你自由发挥，追求现代、简洁、高品质的移动端设计
> 3. 适配手机和平板两种屏幕尺寸
> 4. 遵循 Material Design 3 设计语言，但鼓励创新
>
> **需要设计的界面列表**（请逐一设计，确保风格一致）：
>
> **认证流程：**
> 1. **启动加载屏 (Splash Screen)** — 品牌 Logo 展示，简洁过渡
> 2. **生物识别锁定屏** — 模糊背景遮罩 + 锁定图标 + "点击解锁" 按钮
> 3. **登录页** — 用户名/密码表单 + 品牌展示 + 可折叠的服务器 URL 配置
>
> **主导航（底部 5 Tab）：首页 / 图库 / 文件夹 / 收藏 / 设置**
>
> **首页 Tab：**
> 4. **首页** — 大幅 Hero 轮播区（支持自动轮播的沉浸式大图/视频）+ 下方"最近添加"水平滚动小卡片列表 + 下拉刷新
>
> **图库 Tab：**
> 5. **图库页** — 顶部搜索栏 + 排序/布局切换工具 + 媒体网格（支持等高网格和瀑布流两种布局切换）+ 无限滚动 + 骨架屏加载态
>
> **文件夹 Tab：**
> 6. **文件夹页（根目录）** — 文件夹卡片网格（显示文件夹名 + 媒体数量）+ 媒体网格
> 7. **文件夹页（子目录）** — 同上但带返回按钮，标题为当前目录名
>
> **收藏 Tab：**
> 8. **收藏页** — 分区展示：顶部"收藏的文件夹"卡片 + 下方"收藏的媒体"网格 + 空状态设计
>
> **媒体查看器（全屏覆盖层）：**
> 9. **图片查看模式** — 全屏图片 + 双指缩放 + 双击放大 + 底部渐变控制栏
> 10. **视频播放模式** — 全屏视频 + 底部控制栏（播放/暂停/进度条/时间/旋转）
> 11. **音频播放模式** — 沉浸式背景 + 大面积专辑封面占位 + 毛玻璃控制面板（进度条 + 上/暂停/下）
> 12. **信息面板** — 底部上滑半透明面板，展示文件名/大小/分辨率/路径/网络速率/EXIF 相机信息
>
> **浮层与弹窗：**
> 13. **迷你播放器** — 右上角浮动胶囊，显示歌曲名 + 播放/暂停 + 关闭 + 进度条
> 14. **操作菜单 (Bottom Sheet)** — 长按媒体/文件夹弹出：收藏切换 + 永久删除
> 15. **确认对话框** — 居中弹窗，图标 + 标题 + 描述 + 双按钮
>
> **设置页（双 Tab 切换）：**
> 16. **APP 设置 Tab** — 语言切换 / 主题三选一（跟随系统/浅色/深色）/ 轮播配置（来源选择 + 开关）/ 生物识别开关 / 缓存管理 / 用户信息 + 退出登录
> 17. **服务器管理 Tab** — 系统统计卡片（4 张）/ 4 个后台维护任务行（带进度条和暂停/停止控制）/ 并发数调节器 / 用户管理列表 + 用户编辑 Modal
>
> **全局组件：**
> 18. **Toast 通知** — 浮动通知条（成功/错误/信息/进度）
> 19. **骨架屏** — 加载占位动画组件
>
> 请为每个界面生成高保真设计稿，确保：
> - 所有界面的视觉风格统一（配色、字体、圆角、间距一致）
> - 交互状态完整（正常态、按压态、禁用态、加载态、空状态）
> - 深浅双主题各一套
> - 手机和平板两种布局适配
